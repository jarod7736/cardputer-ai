# M2 — Proxy MVP (claude-proxy on lobsterboy) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the smallest possible **OAI-compatible Chat Completions** HTTP service on lobsterboy that translates a single Anthropic Messages API upstream into the wire format the Cardputer firmware will speak. Output: from any host on the home network (LAN, Tailnet, or WG tunnel via the Cardputer), a `curl` against `http://lobsterboy:8420/v1/chat/completions` with a valid bearer token streams a Claude response back as OAI-shaped SSE chunks.

**Architecture:** A small FastAPI app under a Python venv, running as a dedicated `cardputer-proxy` system user, managed by a systemd unit on lobsterboy. The app exposes one chat endpoint and one health endpoint. A single hard-coded profile (the "claude-opus" profile from DESIGN.md §4.2) is baked into config; profile CRUD is M4. The Anthropic API key is loaded via systemd's `LoadCredential` from a file in `/etc/cardputer-proxy/secrets/` so it never lives in process env vars or service files visible via `systemctl cat`. SSE pass-through is implemented manually with `httpx.AsyncClient` streaming — no Anthropic SDK dependency (keeps surface area small and the translation explicit).

**Tech Stack:**
- Python 3.11+ (lobsterboy already has a recent Ubuntu LTS; verify in Task 1)
- FastAPI + uvicorn (ASGI server)
- `httpx` for async upstream calls with native SSE streaming
- `pydantic` for request/response schemas
- `pyproject.toml` + `uv` (or `pip` if `uv` isn't installed) for dependency management
- systemd unit with `LoadCredential` for Anthropic secret
- Tailscale (already present on lobsterboy) for host reachability; WG (M1) for Cardputer access via the home LAN

**Out of scope for M2:**
- Profile CRUD endpoints (that's M4)
- Multiple provider adapters — OpenAI / Ollama / OpenClaw — (also M4)
- Server-side conversation history (v2)
- TLS termination — the service binds to a Tailscale-only address (`tailscale0` or `0.0.0.0` with firewall scope), and traffic is already encrypted by Tailscale and by WG separately. Public exposure is explicitly not part of M2.
- Per-device bearer-token issuance (M5 provisioning). M2 uses one hard-coded token.
- Reverse-proxy front-end (Caddy/Traefik). DESIGN.md §1 mentions one but it adds nothing for an internal-only service; revisit when we expose anything publicly.

**Carry-overs from M0 + M1:**
- The Cardputer can reach lobsterboy at lobsterboy's LAN address (via WG full-tunnel default route).
- The Cardputer firmware speaks **OpenAI-compatible Chat Completions over SSE** (DESIGN.md §4.1).
- WG tunnel + lobsterboy's Tailscale presence mean we have two independent encrypted-and-authenticated paths to the host. We do not need to add a third (TLS) for M2.

**Verification style:** Python service — fully test-first. Each adapter / handler / auth check gets a unit test using FastAPI's `TestClient`. Integration tests use `pytest-httpx` to mock the Anthropic upstream. The final end-to-end test is a real `curl` from a Tailnet host (and from the Cardputer once M3 wires it up), but M2's done-criteria is satisfied by the curl.

## Decisions to confirm before starting

These have sensible defaults below; flag if any are wrong for your environment:

1. **Listen address & port**: `0.0.0.0:8420`, firewalled so only the Tailnet and the WG subnet can reach it. (Alternative: bind to `tailscale0` specifically — but that excludes the WG path. `0.0.0.0` + `ufw allow from <tailnet>/<wg-subnet>` is simpler.)
2. **Service user**: `cardputer-proxy` (dedicated, non-login).
3. **Install path**: `/opt/cardputer-proxy/` (app), `/etc/cardputer-proxy/` (config), `/var/log/cardputer-proxy/` (logs).
4. **Python**: use the distro's `python3.11+` via a venv at `/opt/cardputer-proxy/.venv` — no PPA, no pyenv.
5. **Secret storage**: systemd `LoadCredential=` reads `/etc/cardputer-proxy/secrets/anthropic_api_key`. File mode `0600`, owned by root. systemd hands the secret to the service process via a runtime path. The API key is never in the unit file, env file, or process env reachable by other users.
6. **Bearer token for M2**: single hard-coded token in the same secrets directory (`/etc/cardputer-proxy/secrets/device_bearer_token`). Per-device tokens are M5.
7. **Profile for M2**: one hard-coded profile in `/etc/cardputer-proxy/config.toml`:
   ```toml
   [[profiles]]
   id = "claude-opus"
   label = "Claude Opus 4.7"
   provider = "anthropic"
   endpoint = "https://api.anthropic.com"
   model = "claude-opus-4-7"
   max_tokens = 4096
   temperature = 1.0
   ```

```python
"""Pydantic models for the device-facing wire protocol (OAI-compatible)."""

from __future__ import annotations

from typing import Literal

from pydantic import BaseModel, Field, field_validator


Role = Literal["system", "user", "assistant"]


class Message(BaseModel):
    role: Role
    content: str


class ChatCompletionRequest(BaseModel):
    profile_id: str = Field(..., min_length=1)
    messages: list[Message] = Field(..., min_length=1)
    stream: bool = True
    max_tokens: int | None = Field(default=None, ge=1, le=200_000)
    temperature: float | None = Field(default=None, ge=0.0, le=2.0)

    @field_validator("messages")
    @classmethod
    def _at_least_one_user_or_assistant(cls, v: list[Message]) -> list[Message]:
        if not any(m.role in ("user", "assistant") for m in v):
            raise ValueError("messages must include at least one user or assistant turn")
        return v


class Profile(BaseModel):
    id: str
    label: str
    provider: Literal["anthropic"]
    endpoint: str
    model: str
    max_tokens: int = 4096
    temperature: float = 1.0
```

```python
"""Runtime config + secret loading.

Secrets and the profile catalog are loaded once at app startup, not on
each request, so swapping a secret requires a service restart. That's an
intentional choice: secret rotation is a rare event and the simplicity
is worth more than hot-reload.
"""

from __future__ import annotations

import os
import tomllib
from dataclasses import dataclass
from pathlib import Path

from cardputer_proxy.schemas import Profile


DEFAULT_PROFILE = Profile(
    id="claude-opus",
    label="Claude Opus 4.7",
    provider="anthropic",
    endpoint="https://api.anthropic.com",
    model="claude-opus-4-7",
    max_tokens=4096,
    temperature=1.0,
)


@dataclass(frozen=True)
class Settings:
    anthropic_api_key: str
    device_bearer_token: str
    profiles: dict[str, Profile]

    def get_profile(self, profile_id: str) -> Profile | None:
        return self.profiles.get(profile_id)


def _read_secret_file(path: Path, name: str) -> str:
    if not path.is_file():
        raise RuntimeError(f"missing secret file: {name} at {path}")
    val = path.read_text(encoding="utf-8").strip()
    if not val:
        raise RuntimeError(f"empty secret file: {name} at {path}")
    return val


def load_settings() -> Settings:
    secret_dir = Path(os.environ.get("CARDPUTER_PROXY_SECRETS_DIR", "/etc/cardputer-proxy/secrets"))
    anthropic_key = _read_secret_file(secret_dir / "anthropic_api_key", "anthropic_api_key")
    bearer = _read_secret_file(secret_dir / "device_bearer_token", "device_bearer_token")

    profiles: dict[str, Profile] = {DEFAULT_PROFILE.id: DEFAULT_PROFILE}
    config_path_raw = os.environ.get("CARDPUTER_PROXY_CONFIG_PATH")
    if config_path_raw:
        config_path = Path(config_path_raw)
        if config_path.is_file():
            data = tomllib.loads(config_path.read_text(encoding="utf-8"))
            profiles = {}
            for entry in data.get("profiles", []):
                p = Profile.model_validate(entry)
                profiles[p.id] = p
            if not profiles:
                profiles = {DEFAULT_PROFILE.id: DEFAULT_PROFILE}

    return Settings(
        anthropic_api_key=anthropic_key,
        device_bearer_token=bearer,
        profiles=profiles,
    )
```

```python
"""Bearer-token authentication dependency.

M2 has a single token; M5 will move to per-device tokens with scopes.
"""

from __future__ import annotations

import hmac

from fastapi import Depends, Header, HTTPException, status

from cardputer_proxy.config import Settings


def _settings_dep() -> Settings:  # overridden in app.py via dependency_overrides
    raise RuntimeError("Settings dependency not wired up")


def require_bearer(
    authorization: str | None = Header(default=None),
    settings: Settings = Depends(_settings_dep),
) -> None:
    """Reject if Authorization is missing or doesn't match the configured token.

    Uses constant-time comparison so we don't leak token length via timing.
    """
    if not authorization or not authorization.lower().startswith("bearer "):
        raise HTTPException(status_code=status.HTTP_401_UNAUTHORIZED, detail="missing bearer")
    presented = authorization.split(" ", 1)[1].strip()
    expected = settings.device_bearer_token
    if not hmac.compare_digest(presented.encode("utf-8"), expected.encode("utf-8")):
        raise HTTPException(status_code=status.HTTP_401_UNAUTHORIZED, detail="invalid bearer")
```

```python
"""FastAPI app factory + routes.

Adapters get wired in here in Task 4; for M2-Task-3 this only covers
auth + a stub /v1/chat/completions that 404s on unknown profile.
"""

from __future__ import annotations

from fastapi import Depends, FastAPI, HTTPException, status

from cardputer_proxy import auth
from cardputer_proxy.config import Settings, load_settings
from cardputer_proxy.schemas import ChatCompletionRequest


def create_app() -> FastAPI:
    settings = load_settings()
    app = FastAPI(title="cardputer-proxy", version="0.1.0")

    # Wire the auth Settings dependency to this app's settings.
    app.dependency_overrides[auth._settings_dep] = lambda: settings

    @app.get("/healthz")
    def healthz() -> dict[str, str]:
        return {"status": "ok"}

    @app.post(
        "/v1/chat/completions",
        dependencies=[Depends(auth.require_bearer)],
    )
    async def chat_completions(req: ChatCompletionRequest):
        profile = settings.get_profile(req.profile_id)
        if profile is None:
            raise HTTPException(status_code=status.HTTP_404_NOT_FOUND,
                                detail=f"unknown profile: {req.profile_id}")
        # The adapter dispatch + SSE streaming lands in Task 4.
        raise HTTPException(status_code=status.HTTP_501_NOT_IMPLEMENTED,
                            detail="anthropic adapter wired up in Task 4")

    return app
```

### Wire-format mapping (reference)

| OAI request field | Anthropic Messages field | Notes |
|---|---|---|
| `messages` with role=system | `system` (string) | Concatenate system messages with `\n\n` separator |
| `messages` with role=user/assistant | `messages` (list) | Pass through; Anthropic requires alternating user/assistant |
| `model` (from profile) | `model` | Map by profile, e.g. `claude-opus-4-7` |
| `max_tokens` (req or profile default) | `max_tokens` | Required by Anthropic; default 4096 |
| `temperature` | `temperature` | Pass through |
| `stream: true` | `stream: true` | Always true from the device for M2 |

| Anthropic SSE event | OAI SSE chunk |
|---|---|
| `message_start` | First chunk with empty `delta.role: "assistant"` |
| `content_block_delta` with `text_delta` | `delta.content: <text>` |
| `message_stop` | `data: [DONE]\n\n` |
| Anything else (tool blocks, etc.) | Dropped for M2 — log only |

```python
"""Adapter protocol — what every provider adapter must implement."""

from __future__ import annotations

from typing import AsyncIterator, Protocol

from cardputer_proxy.schemas import ChatCompletionRequest, Profile


class ChatChunk:
    """An OAI-shaped SSE chunk as a JSON-serializable dict."""

    @staticmethod
    def role(model: str) -> dict:
        return {
            "object": "chat.completion.chunk",
            "model": model,
            "choices": [{"index": 0, "delta": {"role": "assistant"}, "finish_reason": None}],
        }

    @staticmethod
    def text(model: str, text: str) -> dict:
        return {
            "object": "chat.completion.chunk",
            "model": model,
            "choices": [{"index": 0, "delta": {"content": text}, "finish_reason": None}],
        }

    @staticmethod
    def done(model: str, reason: str = "stop") -> dict:
        return {
            "object": "chat.completion.chunk",
            "model": model,
            "choices": [{"index": 0, "delta": {}, "finish_reason": reason}],
        }


class Adapter(Protocol):
    async def stream_chat(
        self,
        profile: Profile,
        request: ChatCompletionRequest,
        secret: str,
    ) -> AsyncIterator[dict]:
        """Yield OAI-shaped chunk dicts. The caller wraps them in SSE."""
```

```python
"""Helpers to format OAI-shaped chunks as Server-Sent Events."""

from __future__ import annotations

import json
from typing import AsyncIterator


def to_sse_event(chunk: dict) -> str:
    return f"data: {json.dumps(chunk, separators=(',', ':'))}\n\n"


def done_event() -> str:
    return "data: [DONE]\n\n"


async def chunks_to_sse(chunks: AsyncIterator[dict]) -> AsyncIterator[bytes]:
    """Wrap a chunk iterator into an SSE byte stream, terminated by [DONE]."""
    async for ch in chunks:
        yield to_sse_event(ch).encode("utf-8")
    yield done_event().encode("utf-8")
```

```python
"""Anthropic Messages API adapter.

Translates one OAI Chat Completions request into one Anthropic Messages
request, and re-emits Anthropic's SSE event stream as OAI-shaped chunks.

We do this by hand (no Anthropic SDK) for three reasons:
1. The SDK is heavy for what we need (auth header + JSON POST + SSE read).
2. Doing the translation explicitly documents the wire-format mapping in
   one place a reader can audit.
3. We want fine control over the SSE pass-through and connection
   lifetime, which is awkward through the SDK's higher-level abstractions.
"""

from __future__ import annotations

import json
from typing import AsyncIterator

import httpx

from cardputer_proxy.adapters.base import Adapter, ChatChunk
from cardputer_proxy.schemas import ChatCompletionRequest, Profile


ANTHROPIC_VERSION = "2023-06-01"


def _split_system(req: ChatCompletionRequest) -> tuple[str | None, list[dict]]:
    """Pull system messages out into Anthropic's `system` field."""
    system_parts: list[str] = []
    messages: list[dict] = []
    for m in req.messages:
        if m.role == "system":
            system_parts.append(m.content)
        else:
            messages.append({"role": m.role, "content": m.content})
    system = "\n\n".join(system_parts) if system_parts else None
    return system, messages


class AnthropicAdapter:
    async def stream_chat(
        self,
        profile: Profile,
        request: ChatCompletionRequest,
        secret: str,
    ) -> AsyncIterator[dict]:
        system, messages = _split_system(request)
        body: dict = {
            "model": profile.model,
            "messages": messages,
            "stream": True,
            "max_tokens": request.max_tokens or profile.max_tokens,
        }
        if system is not None:
            body["system"] = system
        if request.temperature is not None:
            body["temperature"] = request.temperature
        elif profile.temperature is not None:
            body["temperature"] = profile.temperature

        headers = {
            "x-api-key": secret,
            "anthropic-version": ANTHROPIC_VERSION,
            "content-type": "application/json",
            "accept": "text/event-stream",
        }

        url = profile.endpoint.rstrip("/") + "/v1/messages"
        emitted_role = False
        async with httpx.AsyncClient(timeout=httpx.Timeout(60.0, read=300.0)) as cx:
            async with cx.stream("POST", url, headers=headers, json=body) as resp:
                resp.raise_for_status()
                async for line in resp.aiter_lines():
                    if not line.startswith("data: "):
                        continue
                    payload = line[len("data: "):]
                    if payload == "[DONE]":
                        break
                    try:
                        evt = json.loads(payload)
                    except json.JSONDecodeError:
                        continue

                    ev_type = evt.get("type")
                    if ev_type == "message_start" and not emitted_role:
                        emitted_role = True
                        yield ChatChunk.role(profile.model)
                    elif ev_type == "content_block_delta":
                        delta = evt.get("delta") or {}
                        if delta.get("type") == "text_delta":
                            text = delta.get("text") or ""
                            if text:
                                yield ChatChunk.text(profile.model, text)
                    elif ev_type == "message_stop":
                        yield ChatChunk.done(profile.model, reason="stop")
                        break
                    # All other event types (content_block_start/stop,
                    # message_delta with usage info, tool blocks) are
                    # intentionally dropped for M2.

    # Helps pyright understand this satisfies the Protocol
    __protocol__ = Adapter
```

## Done criteria

M2 is complete when **all** of these are true:

1. `cd proxy && .venv/bin/pytest -q` passes (≥10 tests across schemas, auth, adapter, passthrough, errors).
2. `cardputer-proxy.service` is `active (running)` on lobsterboy and survives a `systemctl restart`.
3. A `curl` from any Tailnet host (or WG-attached device) against `http://lobsterboy:8420/v1/chat/completions` with the bearer token streams a real Claude response.
4. The Anthropic API key never appears in `systemctl cat cardputer-proxy.service`, any committed file, or any process listing (`ps auxe`).
5. `docs/m2-proxy-report.md` exists with the deployment specifics and the sample curl proof.
6. Repository tag `m2-complete` exists on origin/main.

At that point the device has a real proxy to point at, and M3 (device chat MVP) can begin.
